# Suton v0.2.0 总验收清单

本清单用于 v0.2.0 实现、验证和发布前建立统一完成标准。任何一项无法取得真实证据时，结论只能是阻塞，不能写成已完成。

## 1. 范围与语义

- 已读取 `AGENTS.md`、`.agents/skills/suton-spec-gate/SKILL.md`、`docs/spec/README.md`、`docs/spec/v0.2.0/README.md` 和所有 `docs/spec/v0.2.0/items/*.md`。
- 所有 v0.2.0 条目状态、目标行为、非目标、验收标准和验证矩阵一致。
- v0.2.0 不引入 OCR、自动切题、AI 长篇讲解、知识图谱、覆盖分析、引用完整性、APK、账号体系或生产运维。
- v0.2.0 必须彻底重构前端，不允许旧前端设计影响最终体验。

## 2. 视觉与交互

- 前端符合 Nature 论文式高级浅色自然系：低饱和、清爽、克制、学术产品气质。
- 页面不使用廉价渐变、厚重暗色、花哨紫蓝、卡片堆叠、默认模板感。
- 前端参考成熟产品的信息密度、留白、排版、导航和状态处理方法，但没有复制特定产品的布局、文案、图标组合、品牌元素或可识别视觉资产。
- 主要布局为固定工作台，不因列表内容无限拉长整页。
- 主要页面在 1440px、1280px、1024px、390px 宽度下截图无明显错位、遮挡、溢出。
- 空状态、加载状态、失败状态、无可靠来源状态均经过设计。
- 人工审美审计结论为通过。

## 3. 项目管理

- 新建项目必须由用户输入名称。
- 项目名称不能为空。
- 同一工作区内项目名称唯一。
- 项目可以重命名。
- 项目可以删除，且删除前有二次确认。
- 删除项目采用硬删除：项目、资料、页面文本、chunk、embedding、题目、来源结果和上传文件一并删除；任一文件删除失败时项目删除失败并显示错误。
- 项目列表在固定区域内滚动，不撑长页面。

## 4. 资料管理

- PDF 可以删除。
- PDF 可以重新处理。
- 删除 PDF 采用硬删除：资料记录、页面文本、chunk、embedding、关联来源结果和上传文件一并删除；题目记录保留但不再展示该资料来源。
- 资料列表在固定区域内滚动，不撑长页面。
- 资料详情展示页数、chunk 数、处理状态、失败原因和最近处理时间。
- 检索范围可以按资料文件筛选。

## 5. 上传与处理进度

- 上传过程展示“纸页入库”视觉组件，而不是只切换状态文字。
- 浏览器能提供上传字节进度时，使用真实上传进度驱动细线填充；不能提供时，使用非百分比流动线，不显示伪百分比。
- 后台处理展示可视化处理轨道：上传完成、提取文字、切块、生成 embedding、建立索引、完成或失败。
- 当前处理节点有轻量动效，已完成节点和失败节点有明确视觉状态。
- 处理失败展示可理解原因。
- 用户刷新页面后仍能看到真实处理状态。

## 6. 题目与来源阅读

- 题目历史可见。
- 每道题保存最近一次检索结果；重新检索会替换该题当前可见结果。
- 检索中、检索失败、无可靠来源均有清晰状态。
- 点击来源结果后进入 PDF 详情视图。
- PDF 详情视图可以跳转目标页，并展示命中段落和上下文。
- 无文件 ID、文件名、页码、chunk ID、原文片段、上下文、PDF 页入口、pgvector 相似度分数、排序位置和确定性命中原因的结果不得展示。

## 7. 验证与发布

- `make env-info` 通过并保留输出。
- `make reset-demo`、`make migrate`、`make verify-e2e`、`make test` 通过。
- `make verify-spec` 和 `make verify-secrets` 通过；`make verify-spec` 必须覆盖 v0.2.0 spec 结构检查。
- 视觉截图和人工审美审计记录已归档。
- 修改已追踪文件后已完成 subagent 严格审计。
- 审计通过后已提交 Git commit。
