# 手动输入题目

- 类型：功能新增
- 状态：草案
- 背景：v0.1.0 要验证题目到资料依据的核心价值，最快方式是让用户手动输入或粘贴一道题。
- 当前问题：如果直接做试题 PDF 上传和自动切题，会把核心检索验证拖入复杂文档识别问题。
- 目标行为：用户可以在项目内手动输入一道题，系统保存题目记录，并触发资料依据检索。
- 非目标：不包含试题 PDF 上传、自动切题、题目截图、题号自动识别、批量题目管理。
- 发布必要性：必须发布
- 用户可见影响：用户可以输入题目文本并点击查找资料依据。
- 涉及模块：前端题目输入页、后端题目 API、检索触发逻辑、数据库。
- 配置、接口或数据结构变化：新增 `questions` 数据对象；新增题目创建和检索触发接口。
- 兼容性要求：无特殊要求。
- 验收标准：
  - 用户可以输入非空题目文本。
  - 空题目不能提交。
  - 提交后系统保存题目记录。
  - 提交后触发资料依据检索。
  - 页面能展示检索中、成功和无结果状态。
- 验证矩阵：
