# UI 参考图风格一致性验收

- 类型：UI 验收标准
- 状态：已完成
- 背景：v0.1.0 已完成最小资料溯源闭环，但当前前端只满足信息区块和功能入口要求，视觉风格、布局密度和证据预览体验与用户提供的 UI 参考图差距明显。
- 当前问题：`docs/assets/ui/suton-trace-request-ui-draft.png` 只在 `README.md` 中被定义为视觉和信息架构参考，没有形成可执行的 UI 验收标准，导致当前前端即使不像参考图也可以通过 v0.1.0 spec。
- 目标行为：v0.1.0 前端必须在不引入已排除功能主链路的前提下，明显贴近参考图的视觉语言和核心信息架构，让用户第一眼能识别为同一产品方向的资料溯源界面。
- 非目标：不包含知识图谱真实算法、覆盖分析真实计算、引用完整性真实分析、复杂统计图表、移动端专项适配、AI 长篇讲解、自动切题、OCR 和 bbox 高亮。
- 发布必要性：必须发布
- 用户可见影响：用户打开 Web App 后应看到接近参考图的信息层级、左侧导航、中心溯源请求工作区、右侧证据预览和资料依据卡片，而不是通用表单式原型。
- 涉及模块：前端页面、前端样式、端到端截图验证、文档验收记录。
- 配置、接口或数据结构变化：无后端接口或数据结构变化；允许前端增加只用于展示已有 v0.1.0 数据的静态装饰层和视觉分组。
- 兼容性要求：不得破坏现有 v0.1.0 API、Playwright 闭环、资料上传、题目输入、来源结果展示和缺来源过滤行为。
- 验收标准：
  - 前端首屏必须采用参考图的三栏信息架构：左侧项目/资料/溯源请求导航，中间溯源请求工作区，右侧证据预览面板。
  - 左侧导航必须包含 Suton 品牌区、项目入口、材料库入口、溯源请求入口、证据审阅入口和设置入口；其中覆盖分析可以作为禁用或弱化入口展示，但不得实现真实覆盖分析功能。
  - 中间工作区必须突出当前请求标题、请求状态、仅基于已上传资料的可信提示，并以可视化方式呈现当前题目和来源候选关系；可以使用静态布局、节点或列表模拟参考图的溯源视图，但不得声称具备知识图谱或覆盖分析能力。
  - 右侧必须提供证据预览面板，展示资料名称、页码、片段锚点、来源片段、相似度分数、命中原因和 PDF 页码入口。
  - 页面必须保留资料库入口、资料上传、手动题目输入、资料依据结果展示和空结果状态。
  - 视觉风格必须贴近参考图：浅色纸感背景、青绿色主色、低饱和分隔线、紧凑信息密度、圆形或节点化证据元素、右侧白色证据面板、底部已上传资料条或等价资料概览。
  - 页面不得退化为通用表单、普通仪表盘卡片堆叠或与参考图无关的营销页。
  - Playwright 必须至少覆盖首屏关键 UI 区域存在性，包含左侧导航、中间溯源工作区、右侧证据预览、资料上传入口、题目输入入口和来源结果卡片。
  - 必须保留现有真实 Web、真实后端、真实 PostgreSQL/pgvector、真实 Redis/RQ 和真实 embedding 闭环验证。
